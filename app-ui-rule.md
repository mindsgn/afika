# App UI Rules (Fintech)

**Purpose**
Deliver a clean, high‑trust interface with strong hierarchy, readable numbers, and consistent action styling.

**Hierarchy & Layout**
- Primary actions are visually dominant and consistent across screens.
- Keep card layouts simple and scannable.
- Group related information; avoid walls of text.
- Favor vertical rhythm and consistent spacing scale.

**Typography**
- Use a clear numeric style for amounts.
- Emphasize totals and balances with size/weight, not color alone.
- Keep secondary text lighter and smaller, but always readable.

**Color & States**
- Use a restrained palette with clear intent.
- Primary action color is reserved for the main CTA.
- Success, warning, error colors must be consistent everywhere.
- Disabled states must be visually obvious and non‑interactive.

**Buttons & Inputs**
- Primary buttons: filled, full‑width where possible, with haptics.
- Secondary buttons: outline or text-only, with haptics.
- Inputs: clear focus state, inline errors, and helper text when needed.

**Data Presentation**
- Always format currency with symbol and grouping.
- Use subtle dividers or spacing between transaction list items.
- Show skeletons for loading lists, not spinners inside lists.

**Motion**
- Use short, purposeful transitions for state changes.
- Avoid heavy animations in critical flows.

**Iconography**
- Use simple, consistent icons for status and actions.
- Avoid decorative icons near amounts or confirmations.
